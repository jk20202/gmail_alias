API 接口文档

所有接口(除 /api/login 外)需在 Header 中携带 Authorization: Bearer <token>
GET/health
健康检查,返回服务状态与账户数量(无需鉴权)
# 健康检查 curl http://localhost:8000/health
POST/api/login
登录校验,验证令牌是否正确
字段类型说明
tokenstring管理令牌
# 登录 curl -X POST http://localhost:8000/api/login \ -H "Content-Type: application/json" \ -d '{"token":"__CURRENT_TOKEN__"}'
GET/api/accounts
获取账户列表(token 字段会脱敏显示)
# 获取账户列表 curl http://localhost:8000/api/accounts \ -H "Authorization: Bearer __CURRENT_TOKEN__"
POST/api/accounts
添加账户
字段类型说明
emailstringGmail 邮箱地址
tokenstring应用密码 (App Password)
labelstring账户标签(可选)
# 添加账户 curl -X POST http://localhost:8000/api/accounts \ -H "Authorization: Bearer __CURRENT_TOKEN__" \ -H "Content-Type: application/json" \ -d '{"email":"test@gmail.com","token":"abcd1234efgh5678","label":"测试号"}'
PUT/api/accounts/{account_id}
更新账户信息(email/token/label 均为可选)
# 更新账户 curl -X PUT http://localhost:8000/api/accounts/abcd1234 \ -H "Authorization: Bearer __CURRENT_TOKEN__" \ -H "Content-Type: application/json" \ -d '{"label":"新标签"}'
DELETE/api/accounts/{account_id}
删除账户
# 删除账户 curl -X DELETE http://localhost:8000/api/accounts/abcd1234 \ -H "Authorization: Bearer __CURRENT_TOKEN__"
POST/api/email/fetch
查询邮件。需指定 account_id 或同时提供 email+token
字段类型说明
account_idstring已存账户 ID(二选一)
emailstring邮箱(二选一)
tokenstring应用密码(配合 email)
tostring收件人过滤(IMAP TO)
senderstring发件人过滤(IMAP FROM)
subjectstring主题过滤(IMAP SUBJECT)
bodystring正文过滤(IMAP TEXT)
keywordstring本地关键词过滤(综合)
unseenbooltrue 仅未读 / false 仅已读
start_timestring开始时间 (YYYY-MM-DDTHH:MM:SS)
end_timestring结束时间
limitint返回上限,默认 50
# 查询邮件 curl -X POST http://localhost:8000/api/email/fetch \ -H "Authorization: Bearer __CURRENT_TOKEN__" \ -H "Content-Type: application/json" \ -d '{ "account_id": "abcd1234", "sender": "noreply@example.com", "start_time": "2026-07-07T00:00:00", "end_time": "2026-07-07T23:59:59", "limit": 20 }'
POST/api/email/mark_read
将匹配的邮件标记为已读。可按 sender/subject 过滤,留空则标记全部
字段类型说明
account_idstring已存账户 ID(二选一)
emailstring邮箱(二选一)
tokenstring应用密码(配合 email)
senderstring发件人过滤
subjectstring主题过滤
# 标记已读 curl -X POST http://localhost:8000/api/email/mark_read \ -H "Authorization: Bearer __CURRENT_TOKEN__" \ -H "Content-Type: application/json" \ -d '{ "account_id": "abcd1234", "sender": "noreply@example.com" }'
